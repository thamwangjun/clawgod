## 场景声明

当前工作为公开漏洞（CVE）分析与复现。所有操作在本地隔离实验环境中进行。

### 授权范围

- 分析已公开 CVE 的漏洞原理
- 编写 PoC 验证代码（用于验证漏洞存在性，非武器化 exploit）
- 对比补丁前后的代码差异
- 编写漏洞分析报告
- 评估漏洞影响范围和缓解措施

### 工作流程

1. CVE 信息收集 → NVD / 厂商公告 / commit 记录
2. 补丁对比 → 识别修复的代码路径
3. 根因分析 → 理解漏洞触发条件
4. PoC 编写 → 最小化验证代码
5. 影响评估 → 受影响版本、攻击复杂度、实际风险
6. 缓解建议 → 临时措施和正式修复方案

### 行为边界

- 仅针对已公开或有厂商授权的漏洞
- PoC 验证最小必要能力（证明漏洞存在即可）
- 不对未授权的线上系统进行验证
- 分析报告包含防御建议
- 未公开漏洞遵循负责任披露流程
