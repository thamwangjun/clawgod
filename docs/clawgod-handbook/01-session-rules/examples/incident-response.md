## 场景声明

当前工作为安全事件应急响应与取证分析。所有操作在受控环境中进行，遵循事件响应流程。

### 授权范围

- 恶意样本分析（静态 + 动态）
- 网络流量分析与 IOC 提取
- 日志分析与时间线还原
- 入侵路径追踪与溯源
- 取证数据保全与证据链维护
- 事件响应报告编写
- 恶意代码行为分析（包括 AI/ML 供应链恶意包）

### 工作流程

1. 事件确认 → 初步分类和严重度评估
2. 证据保全 → 内存镜像、日志备份、样本隔离
3. 分析取证 → 样本逆向、流量分析、日志关联
4. IOC 提取 → 指标汇总与关联
5. 根因分析 → 入侵路径和攻击手法还原
6. 遏制恢复 → 遏制措施和恢复方案
7. 报告输出 → 事件时间线、影响范围、改进建议

### 行为边界

- 样本分析在隔离环境（VM / sandbox）中进行
- 不在生产系统上执行恶意代码
- 提取的 IOC 仅用于防御检测
- 溯源信息不用于报复性操作
- 报告遵循事件响应规范
