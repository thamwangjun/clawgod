# External Integrations

**Analysis Date:** 2026-04-30

## APIs & External Services

**Anthropic Claude Code (upstream):**
- Role: The binary being patched — ClawGod wraps and modifies it
- Source: npm registry (`@anthropic-ai/claude-code-<platform>@latest`)
- Fetch mechanism: `npm pack` in a temp dir during install; no SDK used
- API auth: `ANTHROPIC_API_KEY` env var (set via `~/.clawgod/provider.json`)
- Base URL: `ANTHROPIC_BASE_URL` (defaults to `https://api.anthropic.com`, overridable for third-party endpoints)
- Referenced in: `install.sh` lines 182–207, `install.ps1` lines 161–251

**GitHub REST API (unauthenticated, public):**
- Purpose: Live stats on the landing page — star count and total release download count
- Endpoints:
  - `GET https://api.github.com/repos/0Chencc/clawgod` — star count (`stargazers_count`)
  - `GET https://api.github.com/repos/0Chencc/clawgod/releases?per_page=100` — download totals
- Auth: None (60 req/hr per IP anonymous limit)
- Referenced in: `web/src/main.ts` lines 150–168

**GitHub Raw Content (version badge JSON):**
- Purpose: Landing page "Claude tested" pill — shows last daily-CI verified Claude version
- Endpoint: `https://raw.githubusercontent.com/0Chencc/clawgod/badges/claude-version.json`
- Written by: `compat-daily.yml` CI workflow on success (force-pushes to `badges` branch)
- Schema: `{ schemaVersion, label, message, color }` (shields.io endpoint format)
- Referenced in: `web/src/main.ts` line 19, `.github/workflows/compat-daily.yml` lines 141–167

**bun.sh install service:**
- Purpose: Automatic Bun runtime installation if not found on the user's system
- macOS/Linux: `curl -fsSL https://bun.sh/install | bash`
- Windows: `Invoke-Expression "$(Invoke-RestMethod https://bun.sh/install.ps1)"`
- Referenced in: `install.sh` lines 94–101, `install.ps1` lines 96–103

**npm Registry:**
- Purpose: Fetch Anthropic's per-platform Claude Code tarball (`@anthropic-ai/claude-code-<platform>`)
- Protocol: `npm pack` (bash) or a custom Node.js HTTPS fetch script (PowerShell — avoids `npm` dependency)
- Always fetches `@latest` — local binary detection is intentionally skipped to prevent stale-source traps
- Policy note: Documented in `install.sh` lines 138–149 and `install.ps1` lines 144–155

## Data Storage

**Databases:**
- None

**File Storage:**
- Local filesystem only — all state lives under `~/.clawgod/`
  - `~/.clawgod/cli.cjs` — Bun wrapper (generated by installer)
  - `~/.clawgod/cli.original.cjs` — Post-processed + patched Claude Code JS
  - `~/.clawgod/cli.original.cjs.bak` — Backup before patching
  - `~/.clawgod/patch.mjs` — Universal patcher script
  - `~/.clawgod/extract-natives.mjs` — Binary extractor (Mach-O / ELF / PE parser)
  - `~/.clawgod/post-process.mjs` — Rewrites bunfs paths and CJS wrapper invocation
  - `~/.clawgod/repatch.mjs` — Re-extract + re-patch helper for version upgrades
  - `~/.clawgod/provider.json` — User API config (apiKey, baseURL, model, timeoutMs)
  - `~/.clawgod/features.json` — GrowthBook feature flag overrides
  - `~/.clawgod/.source-version` — Installed Claude version stamp
  - `~/.clawgod/vendor/` — Extracted native `.node` modules (NAPI bindings from Anthropic binary)

**Caching:**
- None (application-level). npm's HTTP cache (`~/.npm/_cacache`) used in CI via `actions/cache@v4`.

## Authentication & Identity

**Auth Provider:**
- Anthropic API auth via API key (`ANTHROPIC_API_KEY`) injected by the wrapper at runtime
- Key source: `~/.clawgod/provider.json` — user-managed, not committed
- Third-party API support: if `baseURL` does not match `anthropic.com`, also sets `ANTHROPIC_AUTH_TOKEN` to the same key

**No OAuth, no session management, no identity service** — authentication is fully delegated to the upstream Claude Code binary using environment variable injection.

## Monitoring & Observability

**Error Tracking:**
- None (no Sentry, Datadog, etc.)

**Logs:**
- Console output only (install scripts print colored status lines to stdout/stderr)
- CI logs captured to `/tmp/install.log` during `compat-daily.yml` and asserted against

**CI failure alerting:**
- GitHub Issues: `compat-daily.yml` auto-opens/updates an issue with label `compat-broken` on scheduled run failure
- Implementation: `actions/github-script@v7` in `.github/workflows/compat-daily.yml` lines 169–212

## CI/CD & Deployment

**Hosting:**
- GitHub Pages — single `index.html` at repo root, served at `clawgod.0chen.cc` (CNAME file present)
- No server-side infrastructure — fully static

**Release pipeline (`.github/workflows/release.yml`):**
- Triggered on `v*` tags
- Builds release notes from git log
- Uploads `install.sh` and `install.ps1` as release assets via `gh release create`
- Actions pinned to Node 24 (`FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true`)

**Daily compatibility CI (`.github/workflows/compat-daily.yml`):**
- Runs at 07:17 UTC daily (+ on push to main, PRs touching `install.sh`)
- Platform: `ubuntu-latest` only (macOS/Windows runners cost-prohibitive for daily)
- Steps: checkout → Node 24 → Bun canary → ripgrep → run `install.sh` → verify artifacts → assert 0 failed patches → `claude --version` smoke test → publish version badge to `badges` branch
- Caches: `~/.npm` via `actions/cache@v4`

**Cache cleanup (`.github/workflows/cache-cleanup-weekly.yml`):**
- Runs weekly (details in workflow file)

## Webhooks & Callbacks

**Incoming:**
- None

**Outgoing:**
- `claude update` command inside the patched binary is redirected to call back to clawgod's own installer:
  - macOS/Linux: `curl -fsSL https://github.com/0Chencc/clawgod/releases/latest/download/install.sh | bash`
  - Windows: PowerShell fetches `install.ps1` from GitHub releases via encoded command
  - Implemented as a regex patch in `patch.mjs` (`"Redirect claude update to clawgod self-update"` patch)

## Environment Configuration

**Required for runtime (user system):**
- `ANTHROPIC_API_KEY` — optional if using Anthropic's native auth flow; required for third-party endpoints
- `ANTHROPIC_BASE_URL` — optional; defaults to `https://api.anthropic.com`

**Always injected by wrapper:**
- `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`
- `DISABLE_INSTALLATION_CHECKS=1`
- `USE_BUILTIN_RIPGREP=1`
- `API_TIMEOUT_MS=3000000`
- `CLAUDE_INTERNAL_FC_OVERRIDES` (if `features.json` present)

**Secrets location:**
- No secrets stored in repository
- User API keys live exclusively in `~/.clawgod/provider.json` (local, never committed)
- CI uses only `secrets.GITHUB_TOKEN` (standard Actions token for releases and badge push)
